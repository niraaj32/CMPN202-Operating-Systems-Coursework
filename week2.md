## Objective
The objective of Week 2 was to understand system planning concepts and to select an appropriate Linux distribution based on system requirements, usability, and intended use cases.

## System Planning Considerations
System planning is an important step before installing an operating system. It involves analysing hardware resources, user requirements, and system purpose. Key factors considered during system planning include:

- CPU and memory availability
- Storage capacity
- Security requirements
- Ease of maintenance and updates
- Community and long-term support

Proper system planning ensures that the selected operating system performs efficiently and meets user needs.

## Linux Distribution Selection
Linux offers a wide range of distributions, each designed for different purposes. Some popular Linux distributions include:

- Ubuntu
- Fedora
- Debian
- Arch Linux
- CentOS

Each distribution differs in terms of stability, update frequency, user experience, and support.


## Justification for Selecting Ubuntu
Ubuntu was selected for this coursework due to the following reasons:

- User-friendly interface suitable for beginners
- Long Term Support (LTS) releases offering stability
- Strong community and extensive documentation
- Regular security updates
- Compatibility with Oracle VirtualBox

Ubuntu is widely used in both academic and professional environments, making it an ideal choice for learning operating system concepts.


## Conclusion
This week provided an understanding of system planning and the importance of selecting the right operating system. Ubuntu was chosen as the most suitable Linux distribution due to its stability, support, and ease of use, making it appropriate for coursework and practical learning.
